## Installation

<RegistryInstall name="synergy/orb" client:load />

## Usage

```tsx
import { Orb } from "@/components/ui/synergy/orb"

export default function App() {
  return (
    <div style={{ width: 300, height: 300 }}>
      <Orb />
    </div>
  )
}
```

## Examples

### Custom Colors

Specify gradient colors via the `colors` prop.

<div  className="flex py-4 w-full">
  <Orb colors={["#ffb84b", "#4975d6", "#79b853"]} client:load />
</div>

```tsx
<Orb colors={["#FF6B6B"]} />
<Orb colors={["#FF6B6B", "#4ECDC4"]} />
<Orb colors={["#FF6B6B", "#4ECDC4", "#FFFFFF"]} />
```

### Agent State

Manage visual states to reflect the agent's current activity.

```tsx
<Orb agentState="idle" />
<Orb agentState="thinking" />
<Orb agentState="listening" />
<Orb agentState="talking" />
```

### Manual Volume Control

Directly control input and output volume values.

<div  className="flex py-4 w-full">
  <Orb inputVolume={0.5} outputVolume={0.8} colors={["#FF6B6B"]} client:load />
</div>

```tsx
<Orb inputVolume={0.5} outputVolume={0.8} />
```

### Custom Seed

Use a fixed seed for consistent animation patterns across renders.

<div  className="flex py-4 w-full">
  <Orb seed={1} colors={["#4975d6"]} client:load />
  <Orb seed={2} colors={["#4975d6"]} client:load />
</div>

```tsx
<Orb seed={42} />
```

### Audio Reactivity

Use callback functions to provide real-time volume data.

```tsx
<Orb
  getInputVolume={() => microphoneLevel}
  getOutputVolume={() => speakerLevel}
/>
```

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `colors` | `[string] \| [string, string] \| [string, string, string]` | `["#CADCFC", "#A0B9D1", "#FFFFFF"]` | Color ramp. 1 color: `[c, c, white]`. 2 colors: `[c1, c1, c2]`. 3 colors: as-is. |
| `seed` | `number` | Random | Seed for consistent animation patterns. |
| `agentState` | `AgentState` | `"idle"` | Visual state of the agent. |
| `inputVolume` | `number` | — | Input volume (0–1). Providing any volume prop switches to manual mode. |
| `outputVolume` | `number` | — | Output volume (0–1). Providing any volume prop switches to manual mode. |
| `getInputVolume` | `() => number` | — | Callback to get current input volume. Switches to manual mode. |
| `getOutputVolume` | `() => number` | — | Callback to get current output volume. Switches to manual mode. |
| `className` | `string` | `"relative h-full w-full"` | Custom CSS class for the container. |

### AgentState

```ts
type AgentState = null | "thinking" | "listening" | "talking"
```

| State | Description |
|-------|-------------|
| `null` | Idle state with subtle ambient animation. |
| `"thinking"` | Gentle wandering motion indicating processing. |
| `"listening"` | Responsive animation simulating active listening. |
| `"talking"` | Dynamic animation simulating speech output. |

## Notes

The component uses raw **WebGL** shaders — no Three.js dependency required.

The canvas automatically handles **device pixel ratio** for sharp rendering on high-DPI displays.

The component properly cleans up WebGL resources on unmount.

<Aside type="caution" title="Important Limitations & Details">
Browsers strictly limit the number of active WebGL contexts per page (typically 8 to 16). Be mindful of this limit when rendering multiple instances of the `<Orb />` component simultaneously. If you hit the limit, older contexts will be dropped.
</Aside>
